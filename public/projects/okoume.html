Okoume Redesign (Enedis)

User workflows • Accessibility (RGAA) • Workshops

Context

Okoume is Enedis’s core tool for managing network-quality data (Linky, PME-PMI, contracts). It suffered from data overload, a complex menu, and non-compliant visuals, hurting adoption.

Objectives

• Simplify workflows for smoother, faster use
• Align UI with RGAA standards and Enedis’s brand
• Pinpoint and solve key pain points

Process

• Kick-off workshop with stakeholders (PO, designers, devs)
• User interviews (8 agents) using TEDW method
• Co-creation sessions & tree tests to map menu flows
• Two rounds of prototyping and usability tests

Results

• Workflows simplified and streamlined
• Full RGAA compliance and updated visuals
• Positive feedback on navigation clarity
• Ready-to-ship UX deliverables (audit, prototypes, IA maps)
